CBIE Analytics

Transformar a base de dados do CBIE em uma infraestrutura organizada, automatizada e acessível via plataforma web.

Diagnóstico

Onde o CBIE está hoje

O Problema

  • Dados espalhados em dezenas de planilhas Excel isoladas, sem versionamento.
  • PDFs e relatórios extraídos manualmente, consumindo horas da equipe toda semana.
  • Nenhuma automação de ingestão — cada atualização de dados é um processo manual.
  • Ausência de um banco de dados relacional centralizado e consultável.

O  I M P A C T O

  • Risco alto de erro humano em cada etapa de coleta e transformação de dados.
  • + Horas mensais gastas em tarefas operacionais repetitivas.
  • Impossibilidade de cruzar dados em tempo real para tomada de decisão.
  • Limitação de crescimento: a operação não escala sem infraestrutura de dados.

Solução - Escopo Técnico

MVP: Os 4 pilares da entrega para a CBIE

1. Estruturação da Base de Dados

Auditoria, modelagem e migração de todas as fontes (Excel, CSV, PDF) para um banco PostgreSQL relacional otimizado.

Clique para ver as etapas
Etapa 1.1

Auditoria e Inventário

Levantamento de todas as fontes (Excel, CSV, PDF), catalogação por tipo e frequência de atualização.

Etapa 1.2

Modelagem PostgreSQL

Design do schema relacional com tabelas: defasagem, PLD, produção, preços, pautas, relatórios.

Etapa 1.3

Migração e Carga

Scripts ETL, limpeza de dados, validação cruzada. Mínimo 12 meses de histórico migrado.

2. Automação de Ingestão (N8N + Python)

Workflows automatizados para ler, validar, transformar e injetar dados de defasagem, PLD e pautas no banco sem intervenção humana.

Clique para ver as etapas
Etapa 2.1

Extração e Conexão de Fontes

Monitoramento automático de pastas, consumo de APIs externas e Webhooks para captura de planilhas e PDFs (defasagem, mercado e pautas) sem intervenção humana.

Etapa 2.2

Sanitização e Validação (Data Quality)

Inspeção do dado bruto, padronização de formatos, conversão de valores e bloqueio automático de arquivos corrompidos ou fora do padrão antes da injeção.

Etapa 2.3

Carga Estruturada e Telemetria

Injeção dos dados limpos no PostgreSQL usando protocolos inteligentes (Upsert) para evitar duplicação, seguida de disparo de alertas automáticos (sucesso/erro) para a equipe.

3. Plataforma MVP Web

Dashboard interativo com login, painel admin, gráficos de séries históricas e biblioteca de relatórios em PDF.

Clique para ver as etapas
Etapa 3.1

Autenticação e Gestão (Admin & Auth)

Sistema de login seguro com controle de acesso baseado em níveis (Admin/Cliente). Inclui painel administrativo para gestão de usuários, permissões e monitoramento do status das automações.

Etapa 3.2

Dashboard e Analytics (Séries Históricas)

Interface interativa de alto desempenho com KPIs e gráficos dinâmicos (defasagem, PLD, Brent). Acesso instantâneo a séries históricas completas com filtros avançados e exportação de dados estruturados.

Etapa 3.3

Acervo Inteligente (Biblioteca de Relatórios)

Repositório centralizado para os documentos gerados. Busca otimizada, filtros precisos por setor e download ágil de PDFs, substituindo o envio manual por uma experiência on-demand.

4. Estudo de Marca e UX/UI

Benchmarking de mercado, wireframes e protótipo navegável no Figma alinhado à identidade visual do CBIE.

Clique para ver as etapas
Etapa 4.1

Benchmarking

Análise de 5 concorrentes e da Argus Direct como referência global, com foco em identificar oportunidades de diferenciação para o CBIE.

Etapa 4.2

Identidade da Plataforma

Criação da paleta de cores para dashboards, tipografia para dados e componentes visuais, garantindo alinhamento com a marca existente do CBIE.

Etapa 4.3

Wireframes + Protótipo

Desenvolvimento de wireframes de baixa e alta fidelidade no Figma, incluindo testes com utilizadores reais antes do início da fase de código.

Sem 3

Automação & UX Final

Sem 4

Desenvolvimento Core I

Sem 5

Desenvolvimento Core II & QA

Sem 6
Milestone: Go-Live 🚀

Homologação & Handoff

Ajustes finais de UI/UX, deploy em produção (Vercel/VPS) e treinamento das equipes com entrega de documentação e código.
